## Charte graphique — SI‑SEP Commune

Ce document centralise les couleurs et règles UI du projet, afin de garder une cohérence visuelle.

### Couleurs principales
- **Bleu principal (Hero / Navbar / fonds premium)**: `#0B2B4A`
  - **Déclinaisons utilisées dans les dégradés (fond hero sobre)**:
    - `#0A2A48`
    - `#082744`
    - `#07223C`
    - `#061B32`
    - `#051428`

### Bleu secondaire (surfaces / cartes “bleues”)
- **Bleu secondaire (moins saturé, pour éviter “trop bleu”)**: `#123554`
  - **Déclinaisons**:
    - `#10314E`
    - `#0D2B46`
    - `#0A243D`

### Accents (or institutionnel)
- **Jaune unique (accent / CTA)**: `#FDB813`

### Neutres (contenu)
- **Fond app clair**: `#F6F7FB`
- **Texte principal (sombre)**: `#0F172A` (≈ slate-900)

### Règles d’usage
- **Bleu principal**: réservé aux zones structurantes (hero, navbar, footer, bandeaux).
- **Bleu secondaire**: pour les surfaces internes du hero/dashboard (cartes, panneaux), afin d’alléger la dominance du bleu principal.
- **Or**: utilisé en accent (CTA, badges importants) — éviter d’en mettre partout.
- **Contraste**: toujours assurer lisibilité (texte blanc sur fond bleu + voile si image/motif).

### Border radius (uniforme)
- **Radius UI standard**: `12px` (peu courbé, cohérent et lisible)
- **Règle**: privilégier une seule famille de coins arrondis sur tout le projet (éviter les `rounded-3xl` très courbés).
